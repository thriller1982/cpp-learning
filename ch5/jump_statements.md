# 跳转语句(jump_statements)

## 核心概念

- 跳转语句会打断原本顺序执行的流程。
- `break` 用来立刻结束最近的一层循环或 `switch`。
- `continue` 用来结束当前这一轮循环，直接进入下一轮。
- `goto` 确实存在，但通常不推荐作为常规控制结构。
- 这类语句的关键不是“语法会不会写”，而是要看清它到底打断了哪一层流程。

## `break` 语句

`break` 的作用是：  
立刻终止当前最近的一层循环或 `switch`。

例如：

```cpp
while (cin >> s) {
    if (s == sought)
        break;
}
```

这里一旦找到目标字符串，就直接跳出 `while`，后面的循环体不再执行。

在 `switch` 里，`break` 也一样重要：

```cpp
switch (ch) {
    case 'a':
        ++aCnt;
        break;
    default:
        break;
}
```

它的作用是阻止程序继续往后贯穿。

## `break` 只影响最近一层

这是很重要的一点。

如果你在嵌套循环里写了 `break`，它只会跳出离自己最近的那一层，不会一下跳出所有循环。

所以看到 `break` 时，先问自己：

- 我现在身处哪一层循环或 `switch`
- 它到底会跳到哪里

## `continue` 语句

`continue` 的作用是：  
结束本次循环剩余部分，直接进入下一轮。

例如：

```cpp
for (int i = 0; i != 10; ++i) {
    if (i % 2 == 1)
        continue;
    cout << i << " ";
}
```

这里的逻辑是：

- 奇数那一轮直接跳过后续输出
- 循环继续往下一轮走

所以最终只会输出偶数。

## `break` 和 `continue` 的区别

可以先把它们明确分开：

- `break`：整层循环结束
- `continue`：只是当前这一轮结束

也就是说：

- `break` 之后，这层循环不再继续
- `continue` 之后，循环还会继续，只是跳过本轮剩余代码

## `goto` 语句

`goto` 可以无条件跳到某个标签：

```cpp
goto end;

// ...

end:
    return;
```

语法上它完全合法，但通常不推荐常规使用。  
原因不是“语言不让你用”，而是它很容易让执行路径变得跳跃、难读、难维护。

大多数普通场景里：

- 用循环
- 用条件语句
- 用函数拆分

都能写出更清楚的结构。

所以对 `goto`，复习时知道两件事就够了：

- 它存在
- 它为什么一般不推荐

## 易错点

- 以为 `break` 能一下跳出多层循环。
- 把 `continue` 误解成“结束整个循环”。
- 在 `switch` 里忘记 `break`，却以为会自动停下。
- 写了跳转语句后，没有重新检查后续哪些语句已经不可达。
- 为了图省事滥用 `goto`，让流程难以维护。
