# 跳转语句(jump_statements)

## 核心概念

- 跳转语句会打断原本顺序执行的流程。
- `break` 用来立刻结束最近的一层循环或 `switch`。
- `continue` 用来结束当前这一轮循环，直接进入下一轮。
- `goto` 语法合法，但通常不适合作为常规控制结构。
- 使用跳转语句时，关键是看清它究竟影响哪一层流程。

## `break` 语句

`break` 用来立刻终止最近的一层循环或 `switch`：

```cpp
while (cin >> s) {
    if (s == sought)
        break;
}
```

这里一旦找到目标字符串，就直接跳出 `while`。

在 `switch` 中，`break` 还用来阻止继续向后贯穿：

```cpp
switch (ch) {
    case 'a':
        ++aCnt;
        break;
    default:
        break;
}
```

## `continue` 语句

`continue` 用来结束当前这一轮循环剩余部分，直接进入下一轮：

```cpp
for (int i = 0; i != 10; ++i) {
    if (i % 2 == 1)
        continue;
    cout << i << " ";
}
```

这里奇数那一轮会直接跳过输出，循环继续往下执行，所以最终只输出偶数。

`break` 和 `continue` 的区别可以概括为：

- `break` 结束整层循环
- `continue` 只结束当前这一轮

## 跳转范围

使用跳转语句时，要先看自己处在哪一层结构中。  
如果在嵌套循环里写了 `break`，它只会跳出最近的一层，不会一次跳出所有循环。

同样，`continue` 只影响当前所在的那层循环，不会让外层循环一起结束。

因此看到跳转语句时，最好先明确两件事：

- 它所在的是哪一层循环或 `switch`
- 它之后哪些语句还会执行，哪些已经不会执行

## `goto` 语句

`goto` 可以无条件跳转到某个标签：

```cpp
goto end;

// ...

end:
    return;
```

它语法上完全合法，但通常不推荐常规使用。  
原因不是语言禁止，而是它很容易让执行路径变得跳跃、难读、难维护。

多数普通场景下，用循环、条件语句或函数拆分，都能写出更清楚的结构。

## 易错点

- 以为 `break` 能一下跳出多层循环。
- 把 `continue` 误解成结束整个循环。
- 在 `switch` 里忘记 `break`，却以为会自动停下。
- 写了跳转语句后，没有重新检查哪些语句已经不可达。
- 为了图省事滥用 `goto`，让流程变得难以维护。
