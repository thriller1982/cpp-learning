# 条件语句(conditional_statements)

## 核心概念

- 条件语句根据条件真假决定程序走哪条分支。
- `if` 适合范围判断和复合条件。
- `switch` 适合离散值匹配。
- `else` 总是和最近的未匹配 `if` 配对。
- `switch` 里最常见的问题是漏写 `break`。

## `if` 语句

最基本的 `if` 形式是：

```cpp
if (condition)
    statement;
else
    statement2;
```

它表示条件为真时执行前一条分支，否则执行后一条分支。

条件表达式不要求原本就是 `bool`，只要它能转换成 `bool` 即可：

```cpp
if (x) {
    do_something();
}
```

多分支判断通常写成 `else if` 链：

```cpp
if (score >= 90) {
    grade = 'A';
} else if (score >= 80) {
    grade = 'B';
} else {
    grade = 'C';
}
```

这类结构适合有先后顺序的范围判断。

## `if` 嵌套与 `else` 配对

嵌套 `if` 时，`else` 会配给最近的未匹配 `if`：

```cpp
if (a)
    if (b)
        foo();
    else
        bar();
```

这里的 `else` 属于内层 `if (b)`，等价于：

```cpp
if (a) {
    if (b)
        foo();
    else
        bar();
}
```

因此只要嵌套关系稍复杂，就应该优先加花括号，把分支结构写清楚。  
即使分支里只有一条语句，写成下面这样通常也更稳妥：

```cpp
if (condition) {
    do_something();
}
```

## `switch` 语句

`switch` 适合对离散值进行匹配：

```cpp
switch (ch) {
    case 'a':
        ++aCnt;
        break;
    case 'e':
        ++eCnt;
        break;
    default:
        break;
}
```

它的执行过程可以理解为：

- 先计算控制表达式
- 跳到匹配的 `case`
- 从该位置开始顺序往下执行

`switch` 的控制表达式必须是整数类型，或能转换成整数的类型。  
常见的有 `int`、`char`、`enum`，而 `string` 不能直接拿来做 `switch`。

## `case`、`break` 与 `default`

`case` 标签必须是常量表达式，例如：

```cpp
case 1:
case 'a':
```

普通变量通常不能直接出现在 `case` 后面，因为分支入口需要在编译时确定。

`switch` 中还要特别注意 `break`：

```cpp
switch (n) {
    case 1:
        cout << "one\n";
    case 2:
        cout << "two\n";
}
```

当 `n == 1` 时，这段代码会连续输出两行，因为程序会从匹配位置继续向下执行，这种现象叫贯穿。

多数情况下应主动写 `break` 来结束当前分支。  
`default` 虽然不是必须的，但通常值得保留，用来兜底未覆盖的情况。

## `if` 与 `switch` 的选择

可以用下面的方式区分它们：

- `if` 适合范围条件和复合逻辑条件
- `switch` 适合有限个离散值的匹配

例如 `score >= 90` 更适合 `if`，而字符是否为 `'a'`、`'e'`、`'i'` 更适合 `switch`。

## 易错点

- 误以为 `else` 会自动配给自己想象中的那个 `if`。
- `if` 分支只有一条语句时不写花括号，后续修改时埋下错误。
- 拿 `string` 直接作为 `switch` 的控制表达式。
- 忘记 `case` 标签必须是常量表达式。
- 在 `switch` 里漏写 `break`，导致意外贯穿。
