# 迭代语句(iteration_statements)

## 核心概念

- 迭代语句用来重复执行一段代码。
- `while` 是先判断、后执行。
- 传统 `for` 适合把初始化、条件和更新放在一起看。
- 范围 `for` 适合顺序遍历容器、数组和字符串。
- `do while` 至少会执行一次。

## `while` 语句

最基本的形式是：

```cpp
while (condition)
    statement;
```

它的执行顺序是：

- 先检查条件
- 条件为真才进入循环体
- 每轮结束后再回去判断

所以如果一开始条件就是假，`while` 一次都不会执行。

这种循环适合：

- 不知道要执行多少次
- 只知道“条件成立就继续”

例如：

```cpp
while (cin >> value) {
    sum += value;
}
```

这里不是在数固定次数，而是在“输入还有效”的前提下不断重复。

## 传统 `for` 语句

传统 `for` 的形式是：

```cpp
for (init; condition; expression)
    statement;
```

可以先把它理解成：

- 先执行初始化
- 再判断条件
- 条件成立就执行循环体
- 然后执行更新表达式
- 再回到条件判断

例如：

```cpp
for (int i = 0; i != 10; ++i) {
    cout << i << endl;
}
```

这类写法适合：

- 有明显循环变量
- 迭代方式固定
- 想把“从哪开始、到哪结束、怎么更新”放在一行里看清楚

## `for` 头里定义变量的作用域

```cpp
for (int i = 0; i != 10; ++i) {
    cout << i << endl;
}

// 这里通常已经不能再使用 i
```

这也是传统 `for` 的一个优点：  
把循环控制变量限制在循环本身内部，不让它泄漏到更大作用域。

## 范围 `for` 语句

现代 C++ 里非常常见的一种写法是：

```cpp
for (auto c : str) {
    cout << c << endl;
}
```

它最适合的场景是：

- 你只是想按顺序处理每个元素
- 你不关心下标
- 你也不想手写迭代器或计数变量

## 范围 `for` 默认会拷贝元素

```cpp
for (auto x : vec) {
    x = 0;
}
```

这里的 `x` 是副本，不是原元素。  
所以这段代码改到的是拷贝，原容器内容不会变。

## 想修改原元素，要用引用

```cpp
for (auto& x : vec) {
    x *= 2;
}
```

这里 `x` 绑定到原元素上，所以修改 `x` 就是在修改原容器。

## 只读遍历常用 `const auto&`

```cpp
for (const auto& x : vec) {
    cout << x << endl;
}
```

这种写法的价值是：

- 不拷贝元素
- 不会误改原元素

如果元素本身比较大，这种写法尤其自然。

## 范围 `for` 也有边界

范围 `for` 很好用，但不是所有遍历场景都最适合它。

如果你经常需要：

- 用下标定位
- 同时访问当前位置编号
- 做跨元素的下标运算

那传统 `for` 往往更合适。

## `do while` 语句

`do while` 的形式是：

```cpp
do {
    statement;
} while (condition);
```

它和 `while` 最大的区别是：

- `while`：先判断，再决定要不要执行
- `do while`：先执行一次，再判断要不要继续

所以 `do while` 至少会执行一次。

这类写法常见于：

- 菜单循环
- 重试输入
- 先做一次，再决定是否继续的场景

## 怎么选循环形式

可以先这样记：

- 不知道次数，只知道条件：`while`
- 有明显计数变量：传统 `for`
- 只是顺序遍历元素：范围 `for`
- 至少要先执行一次：`do while`

## 易错点

- 忘记 `while` 可能一次都不执行。
- 把 `for` 头里定义的变量当成循环外也能用。
- 以为 `for (auto x : vec)` 能修改原元素。
- 明明需要修改原容器，却忘记写成 `auto&`。
- 写 `do while` 时漏掉末尾那个分号。
