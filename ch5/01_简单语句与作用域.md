# 简单语句与作用域(simple_statements_and_scope)

## 核心概念

- 第 5 章的主线是语句和控制流。
- 表达式后面加上分号，就构成一条表达式语句。
- 空语句合法，但非常容易改变原本的控制逻辑。
- 花括号括起来的一组语句叫块，它既能充当一条语句，也会引入新作用域。
- 变量应尽量在最接近使用处定义，把作用域控制到最小。

## 这一章在讲什么

程序默认按顺序一条一条执行。  
第 5 章讨论的是：怎样通过不同语句结构控制执行路径，包括条件分支、循环、跳转以及异常处理。

## 表达式语句

最常见的简单语句就是表达式语句：

```cpp
i = ival + 5;
cout << i;
++i;
```

它们的共同点是：

- 本体是一个表达式
- 末尾跟一个分号

表达式负责完成计算或操作，分号把它变成一条完整语句。

## 空语句

空语句就是单独一个分号：

```cpp
;
```

它什么也不做，但语法上完全合法。  
有时会故意这样写：

```cpp
while (cin >> s && s != sought)
    ;
```

这里循环负责不断读取输入，循环体本身为空。

真正需要警惕的是误写空语句：

```cpp
if (condition);
{
    do_something();
}
```

这个分号已经结束了 `if`，后面的块不再受条件控制，语义也随之改变。

## 复合语句和块作用域

用花括号括起来的一组语句叫复合语句，也常直接叫块：

```cpp
{
    int sum = 0;
    sum += 42;
}
```

块有两个作用：

- 在语法上把多条语句当成一条语句
- 引入新的作用域

因此在 `if`、`while`、`for` 等位置，即使语法只要求一条语句，也可以放一个块进去。

块内部定义的名字通常只在这个块内有效：

```cpp
{
    int i = 42;
}

// 这里已经不能再使用 i
```

## 名字隐藏与作用域控制

内层作用域可以定义与外层同名的变量：

```cpp
int i = 10;

{
    int i = 20;
    cout << i << endl;
}
```

这里内层的 `i` 会隐藏外层的 `i`，两者是不同对象。

实际写代码时，更重要的是尽量缩小变量作用域：

```cpp
for (int i = 0; i != 10; ++i) {
    cout << i << endl;
}
```

这种写法让 `i` 只在循环内部有效，更清楚，也更不容易被误用。

## 易错点

- 把分号多写在 `if` 或 `while` 后面，却没意识到逻辑已经改变。
- 看到空语句就断定一定是笔误，没有结合上下文判断。
- 忘记块会引入新的作用域。
- 不清楚内层同名变量会隐藏外层变量。
- 把变量过早定义到外层，导致作用域比实际需要更大。
