# 简单语句与作用域(simple_statements_and_scope)

## 核心概念

- 第 5 章的主线是语句和控制流。
- 表达式后面加上分号，就构成一条表达式语句。
- 空语句是合法语句，但也是高频陷阱来源。
- 花括号包起来的一组语句叫块，它既能当作一条语句，也会引入新作用域。
- 变量尽量在哪里使用，就在哪里定义。

## 这一章在讲什么

程序默认会按顺序一条一条执行下去。  
但真正写程序时，你经常需要：

- 条件成立时才执行某段代码
- 把一段代码重复执行很多次
- 提前结束循环
- 跳过当前这一轮
- 在出错时中断正常流程

所以这一章本质上是在讲：如何控制程序的执行路径。

## 表达式语句

最常见的简单语句，就是表达式语句。

```cpp
i = ival + 5;
cout << i;
++i;
```

它们共同的特点是：

- 本体是一个表达式
- 末尾跟一个分号

可以先把它记成一句话：

- 表达式负责“做事”
- 分号负责把这件事变成一条完整语句

## 空语句

空语句就是单独一个分号：

```cpp
;
```

它什么也不做，但语法上完全合法。

有时你会故意这样写：

```cpp
while (cin >> s && s != sought)
    ;
```

这段代码的意思不是“忘写循环体”，而是：

- 只要条件成立，就持续读输入
- 循环体本身什么也不做

但空语句也是最容易埋坑的地方。

```cpp
if (condition);
{
    do_something();
}
```

这里分号已经把 `if` 结束掉了，后面的块不再受 `if` 控制。  
这属于经典错误，不是风格问题，而是逻辑已经变了。

## 复合语句和块

用花括号括起来的一组语句，叫复合语句，也常直接叫块。

```cpp
{
    int sum = 0;
    sum += 42;
}
```

块最重要的两个作用是：

- 在语法上把多条语句当成“一条语句”
- 引入一个新的作用域

所以像 `if`、`while`、`for` 这些位置，哪怕原本只要求一条语句，你也完全可以放一个块进去。

## 语句作用域

在块内部定义的名字，通常只在这个块里有效。

```cpp
{
    int i = 42;
}

// 这里已经不能再用 i
```

这就是块作用域最核心的直觉：

- 名字不是“定义过就永远可见”
- 它只在所属作用域里存在

## 内层可以隐藏外层同名变量

```cpp
int i = 10;

{
    int i = 20;
    cout << i << endl;
}
```

这里块内部的 `i` 会暂时把外层那个 `i` 隐藏起来。  
这不是“修改了外层变量”，而是内层重新定义了一个同名对象。

所以遇到同名变量时，先问自己：

- 这是在使用外层对象
- 还是在当前块里重新定义了一个新对象

## 尽量缩小变量作用域

这是 C++ 里很重要的一条风格习惯。

```cpp
for (int i = 0; i != 10; ++i) {
    cout << i << endl;
}
```

这里 `i` 只在这个 `for` 循环里有效。  
这种写法通常比先在外面定义一个 `i` 更清楚，因为：

- 变量只在真正需要的地方出现
- 作用域更小，更不容易被误用

可以先把它记成一句话：

- 越晚定义越好
- 越靠近用途越好
- 越少暴露给外部越好

## 易错点

- 把分号多写在 `if` 或 `while` 后面，却没意识到语义已经变了。
- 看到空语句，就下意识以为一定是笔误。
- 忘记块会引入新作用域。
- 不清楚内层同名变量会隐藏外层变量。
- 把循环控制变量提前定义到外层，结果让作用域比需要的更大。
