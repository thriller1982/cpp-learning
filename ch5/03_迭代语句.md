# 迭代语句(iteration_statements)

## 核心概念

- 迭代语句用来重复执行一段代码。
- `while` 适合“条件成立就继续”的循环。
- 传统 `for` 适合把初始化、条件和更新写在一起。
- 范围 `for` 适合顺序遍历容器、数组和字符串。
- `do while` 至少会执行一次。

## `while` 语句

`while` 的基本形式是：

```cpp
while (condition)
    statement;
```

它的执行顺序是：

- 先检查条件
- 条件为真才执行循环体
- 每轮结束后再回到条件判断

因此如果一开始条件就是假，`while` 一次都不会执行。

```cpp
while (cin >> value) {
    sum += value;
}
```

这类循环适合事先不知道要执行多少次，只知道“条件成立就继续”的场景。

## 传统 `for` 语句

传统 `for` 的形式是：

```cpp
for (init; condition; expression)
    statement;
```

可以把它理解成：

- 先执行初始化
- 再判断条件
- 条件成立时执行循环体
- 然后执行更新表达式

例如：

```cpp
for (int i = 0; i != 10; ++i) {
    cout << i << endl;
}
```

它适合有明显循环变量、迭代方式固定的情况。  
在 `for` 头中定义的变量，作用域通常只到这条 `for` 语句结束为止。

## 范围 `for` 语句

范围 `for` 适合按顺序处理每个元素：

```cpp
for (auto c : str) {
    cout << c << endl;
}
```

这种写法不需要自己维护下标或迭代器，因此在单纯遍历时通常更直接。

需要注意的是，范围 `for` 默认会拷贝元素：

```cpp
for (auto x : vec) {
    x = 0;
}
```

这里修改的是副本，不会影响原容器。

如果要修改原元素，应使用引用：

```cpp
for (auto& x : vec) {
    x *= 2;
}
```

如果只是只读遍历，`const auto&` 通常更合适：

```cpp
for (const auto& x : vec) {
    cout << x << endl;
}
```

## `do while` 语句

`do while` 的形式是：

```cpp
do {
    statement;
} while (condition);
```

它与 `while` 的区别在于先执行一次循环体，再判断是否继续。  
因此 `do while` 至少会执行一次。

它常见于菜单循环、重试输入这类“先做一次，再决定是否继续”的场景。

## 循环形式的选择

常见选择方式如下：

- 不知道次数，只知道条件是否成立，用 `while`
- 有明确的初始化、条件和更新，用传统 `for`
- 只是顺序遍历元素，用范围 `for`
- 至少要先执行一次，用 `do while`

如果需要下标、位置编号或跨元素运算，传统 `for` 往往比范围 `for` 更合适。

## 易错点

- 忘记 `while` 可能一次都不执行。
- 把 `for` 头里定义的变量当成循环外也能用。
- 以为 `for (auto x : vec)` 能修改原元素。
- 需要修改原容器时忘记写成 `auto&`。
- 写 `do while` 时漏掉结尾分号。
