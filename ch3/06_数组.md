# 数组(arrays)

## 核心概念

- 这里说的数组是内置数组，不是 `vector`。
- 内置数组大小固定，一旦定义完成就不能改长度。
- 数组不能整体赋值或拷贝。
- 数组在很多表达式里会自动退化成指向首元素的指针，但数组不等于指针。

## 定义与初始化

```cpp
int arr[10];
int arr2[3] = {1, 2, 3};
int arr3[] = {1, 2, 3};
string words[] = {"one", "two", "three"};
```

这里最重要的限制是：数组大小必须在编译时就能确定。

另外，数组一旦创建好，长度就固定了。它不像 `vector` 那样可以继续增长。

## 数组不能整体赋值

```cpp
int a[3] = {1, 2, 3};
int b[3];

// b = a;
```

这不是“语法小毛病”，而是内置数组本身就不支持这种整体赋值语义。

如果你需要能整体拷贝、整体赋值、大小还能变的序列，通常更应该考虑 `vector`。

## 访问元素与遍历

```cpp
int nums[] = {1, 2, 3};
int first = nums[0];
```

范围 `for` 也能直接用在数组上：

```cpp
for (auto n : nums) {
    // 只读
}

for (auto& n : nums) {
    // 修改原数组元素
}
```

和 `vector` 一样，下标从 `0` 开始；越界访问依然是未定义行为。

## 数组退化与类型推断

```cpp
int ia[] = {0, 1, 2, 3};
int* p = ia;
```

这里 `ia` 在表达式里会退化成指向首元素的指针，所以这行等价于：

```cpp
int* p = &ia[0];
```

但要记清楚：数组不是指针，它只是经常会转换成指针。

标准库还提供了更直接的首尾获取方式：

- 这里的 `begin` 和 `end` 指的是标准库函数 `std::begin` 和 `std::end`
- 它们定义在头文件 `<iterator>` 中
- 写全名时就是 `std::begin(ia)` 和 `std::end(ia)`
- `begin(ia)` 返回首元素位置，`end(ia)` 返回最后一个元素后面的尾后位置

```cpp
auto first = begin(ia);
auto last = end(ia);
```

对这里的 `int ia[]` 来说：

- `first` 的类型是 `int*`
- `last` 的类型也是 `int*`

更具体地说：

- `first` 指向 `ia[0]`
- `last` 指向 `ia` 最后一个元素后面的那个位置

所以 `last` 可以拿来做比较或作为遍历终点，但不能解引用。

再往前走一步，就会遇到数组在类型推断里的表现差异：

```cpp
int arr[10];

auto a = arr;
decltype(arr) b = {0};
```

这里的差异很有代表性：

- `auto a = arr;` 时，`arr` 会退化，所以 `a` 是 `int*`
- `decltype(arr)` 看的是声明类型本身，所以 `b` 仍然是 `int[10]`

这也是“数组不等于指针”的一个高频考点。

## 易错点

- 误以为数组和 `vector` 一样支持整体赋值。
- 把数组退化成指针的现象理解成“数组本来就是指针”。
- 写 `auto x = arr;` 却没意识到类型已经变成指针。
- 访问数组元素时忘记越界不会自动报错。
