# 声明与作用域(declaration_and_scope)

## 核心概念

- 声明告诉编译器“有这个名字”。
- 定义不仅告诉编译器“有这个名字”，还真正创建实体。
- 名字只在自己的作用域内可见。
- 内层同名对象会遮蔽外层对象。

## 声明与定义

```cpp
extern int global_x; // 声明
int global_x = 0;    // 定义
```

区别要抓牢：

- 声明：告诉编译器类型和名字，实体可能在别处
- 定义：真正分配存储并创建对象

特殊情况：

```cpp
extern int global_y = 1;
```

只要带了初值，它本质就是定义，不是纯声明。

## 标识符命名规则

合法示例：

```cpp
int valid_name_1 = 7;
```

要避免的名字：

```cpp
// int 1_invalid_name; // 不能以数字开头
// int __reserved_1;   // 双下划线通常保留给实现
// int _Reserved_2;    // 下划线 + 大写开头也应避免
```

记忆原则：

- 变量名要合法
- 更要避免踩到标准库和编译器的保留命名区域

## 作用域与遮蔽

```cpp
int my_val = 100;

int main() {
    int my_val = 42;
    {
        int my_val = 0;
    }
}
```

这段代码体现的是遮蔽：

- 最里层优先看到自己那一层的 `my_val`
- 外层同名变量仍然存在，只是被挡住了

如果想访问全局同名变量，可以用作用域解析符：

```cpp
::my_val
```

## 具体讲讲 `::`

`::` 叫作用域解析符，用来告诉编译器“应该去哪个作用域里找名字”。

在这一节最常见的场景里，它表示从全局作用域查找：

```cpp
int value = 100; // 全局变量

int main() {
    int value = 42;      // 局部变量，遮蔽全局 value
    int local = value;   // 拿到 42
    int global = ::value; // 拿到 100
}
```

这里要抓住一点：`value` 和 `::value` 不是两个不同名字，而是同一个名字在不同作用域中的查找结果。

- `value` 会先在当前作用域里找
- `::value` 会直接去全局作用域里找

所以，局部变量把全局变量遮住时，`::` 就能帮你明确访问外层那个全局名字。

`::` 也不只用于全局变量，标准库里你会反复见到它：

```cpp
std::cout
std::string
```

这表示 `cout` 和 `string` 这两个名字属于 `std` 这个作用域。

可以先把它记成两个最常见的形式：

- `::name`：去全局作用域找 `name`
- `scope::name`：去指定作用域找 `name`

它和下面两种写法要区分开：

- `obj.member`：访问对象成员
- `ptr->member`：通过指针访问对象成员

`::` 处理的是“名字在哪个作用域里”，不是“对象里面有哪些成员”。

## 易错点

- 以为 `extern` 一定不是定义。
- 在大作用域和小作用域里重复使用相同变量名，导致理解混乱。
- 忘了 `::name` 可以拿到全局作用域中的名字。
