# 指针(pointer)

## 核心概念

- 指针保存的是地址。
- `&` 用于取地址，`*` 在表达式中用于解引用。
- 空指针的现代写法是 `nullptr`。
- 指针声明最容易在“一行声明多个变量”时误读。

## 指针基础

```cpp
int i = 42;
int* p = &i;
```

这里：

- `&i` 取出 `i` 的地址
- `p` 存放这个地址

解引用：

```cpp
*p = 100;
```

这表示“顺着 `p` 里保存的地址，找到那个对象并修改它”。

## 指针类型必须匹配

```cpp
double d = 3.14;
double* pd = &d;
// int* pi = &d; // 错误
```

原因不是“编译器故意为难你”，而是编译器需要依赖类型信息来决定如何解释那块内存。

## 空指针

现代 C++ 推荐写法：

```cpp
int* p = nullptr;
```

旧写法：

```cpp
int* p1 = 0;
int* p2 = NULL;
```

它们历史上都存在，但现代代码里应该统一用 `nullptr`，因为它的“空指针语义”最明确。

## 复合类型声明的坑

最经典误区：

```cpp
int* p1, p2;
```

很多人会误读成“这一行声明了两个指针”，其实不是。

真实情况是：

- `p1` 是 `int*`
- `p2` 是 `int`

为了避免这种坑，项目风格最好固定成：

- 写成 `int* p`
- 一行只声明一个变量

更稳妥的写法：

```cpp
int* p1 = nullptr;
int p2 = 0;
int* p3 = nullptr;
```

## 类型别名简化复杂声明

```cpp
using IntPtr = int*;
IntPtr p1 = nullptr;
IntPtr p2 = nullptr;
```

这有助于避免一长串星号和一行多声明带来的视觉噪音。

## 易错点

- 混淆声明里的 `*` 和表达式里的 `*`。
- 解引用空指针。
- 忘记指针类型必须匹配。
- 写 `int* p1, p2;` 却误以为两个都是指针。
