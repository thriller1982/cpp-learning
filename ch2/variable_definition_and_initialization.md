# 变量定义与初始化(variable_definition_and_initialization)

## 核心概念

- 定义会创建对象并分配存储。
- 初始化发生在对象“出生时”。
- 赋值发生在对象已经存在之后。
- 默认初始化是否安全，取决于对象的存储期和类型。

## 初始化 vs 赋值

```cpp
int a = 10; // 初始化

int b;
b = 10;     // 赋值
```

理解这组代码时，重点不是结果都变成了 `10`，而是时机不同：

- `a` 是“出生时就有值”
- `b` 是“先出生，再改值”

## 四种常见初始化形式

```cpp
int i = 0;   // 拷贝初始化
int j(0);    // 直接初始化
int k{0};    // 列表初始化
int m = {0}; // 列表拷贝初始化
```

这里可以顺手分清前两种写法：`T a = x;` 是拷贝初始化，`T a(x);` 是直接初始化。`=` 在这里仍然表示初始化，不是先定义再赋值。对内置类型，这两种写法通常差别不明显；对类类型，它们的匹配规则可能不同。

其中最值得优先记住的是列表初始化，因为它能帮助你在编译期挡住很多危险窄化。

## 默认初始化的差异

### 1. 全局对象

```cpp
int global_val;
```

全局对象属于静态存储期，若未显式初始化，会自动零初始化。

### 2. `static` 局部对象

```cpp
static int static_val;
```

也会自动零初始化。

### 3. 普通局部内置类型对象

```cpp
int local_val;
```

这里最危险：

- 如果它是内置类型
- 如果你没有给初值
- 如果你直接去读它

那就是未定义行为。

更安全的方式：

```cpp
int safe_local{};
```

## 易错点

- 把定义和初始化当成同一件事。
- 认为所有未初始化变量都会自动变成 `0`。
- 在局部作用域直接读取未初始化的内置类型变量。
- 不理解 `{}` 初始化为什么在现代 C++ 里更常见。
