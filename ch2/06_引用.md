# 引用(reference)

## 核心概念

- 引用是对象的别名。
- 引用必须初始化。
- 引用一旦绑定，就不能改绑。
- `const` 引用的绑定能力比普通引用更强。

## 引用的本质：别名

```cpp
int i = 10;
int& r = i;
```

这里 `r` 不是新对象，而是 `i` 的另一个名字。

```cpp
r = 20;
```

这行代码等价于改 `i` 本体。

## 必须初始化

```cpp
int& r = i;   // 正确
// int& bad_r; // 错误
```

普通引用不能悬空。它一出生就必须认主。

## 不能改绑

```cpp
int i = 20;
int& r = i;
int j = 99;

r = j;
```

很多人第一次看到会误会成“让 `r` 改绑到 `j`”。其实不是。

这行的真实含义是：

- 取出 `j` 的值
- 赋给 `r` 代表的对象
- 也就是赋给 `i`

引用的绑定关系没有改变。

## 普通引用的限制

普通引用通常要求：

- 类型严格匹配
- 绑定到可修改的左值

例如：

```cpp
double d = 3.14;
// int& ref_d = d; // 错误
// int& ref_num = 42; // 错误
```

## `const` 引用的特权

```cpp
const int& r1 = 42;
double d = 3.14;
const int& r2 = d;
```

为什么 `r2` 合法？因为编译器会先生成一个临时 `int`，再让 `r2` 绑定到这个临时对象。

这也意味着：

- `r2` 不是直接绑在 `d` 身上
- 后续你再改 `d`
- `r2` 并不会同步变化

## 易错点

- 把引用当成“自动解引用指针”来理解。
- 误以为 `r = j` 会让引用改绑。
- 不清楚普通引用和 `const` 引用在绑定能力上的区别。
