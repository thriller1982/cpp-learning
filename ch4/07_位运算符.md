# 位运算符(bitwise_operators)

## 核心概念

- 位运算符直接操作二进制位，而不是按真假做逻辑判断。
- 常见位运算符有 `~`、`&`、`|`、`^`、`<<`、`>>`。
- 位运算常见于标志位、掩码和底层数据处理。
- 初学阶段应优先掌握无符号整数上的位运算。

## 常见位运算符

```cpp
unsigned int a = 0b0101;
unsigned int b = 0b0011;

auto x1 = ~a;
auto x2 = a & b;
auto x3 = a | b;
auto x4 = a ^ b;
auto x5 = a << 1;
auto x6 = a >> 1;
```

可以先这样理解：

- `~`：按位取反
- `&`：对应位都为 `1` 才得 `1`
- `|`：对应位有一个是 `1` 就得 `1`
- `^`：对应位不同才得 `1`
- `<<`：整体左移
- `>>`：整体右移

## 位运算不是逻辑运算

这一点必须分清：

- `&` 不是 `&&`
- `|` 不是 `||`

例如：

```cpp
bool a = true;
bool b = false;

bool r1 = a && b;
bool r2 = a & b;
```

它们长得像，但用途和规则不是同一套。  
教学上最重要的是别把符号看串了。

## 位运算的常见用途

位运算最常见的使用场景不是“替代普通算术”，而是处理标志位。

```cpp
unsigned int read_flag = 0b0001;
unsigned int write_flag = 0b0010;
unsigned int mode = read_flag | write_flag;
```

这里 `mode` 表示同时打开两个标志。

## 移位时要谨慎

对无符号整数，移位的直觉通常更稳定：

```cpp
unsigned int value = 1;
value = value << 3;
```

而有符号数做位运算，尤其做移位时，更容易和具体实现细节纠缠在一起。  
所以复习时应优先把注意力放在无符号整数上。

## 易错点

- 把 `&`、`|` 当成 `&&`、`||`。
- 只记住符号长相，却不知道它们是在操作二进制位。
- 在有符号整数上随意做移位，还以为结果一定直观稳定。
- 把位运算写进普通业务逻辑，却没有注释说明掩码含义。
