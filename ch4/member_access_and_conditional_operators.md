# 成员访问与条件运算符(member_access_and_conditional_operators)

## 核心概念

- `.` 用于对象访问成员，`->` 用于指针访问成员。
- `ptr->mem` 本质上等价于 `(*ptr).mem`。
- 三目运算符 `?:` 是一种表达式形式的条件选择。
- 条件运算符适合简短选择，不适合塞进过长的复杂逻辑。

## 成员访问运算符

对象直接访问成员时，用 `.`：

```cpp
string s = "abc";
auto len = s.size();
```

指针访问成员时，用 `->`：

```cpp
string s = "abc";
string* p = &s;
auto len = p->size();
```

这里最应该记住的是：

```cpp
p->size()
```

本质上等价于：

```cpp
(*p).size()
```

括号不能少，因为 `.` 的优先级比 `*` 更高。

## 条件运算符

三目运算符的基本形式是：

```cpp
cond ? expr1 : expr2
```

意思是：

- 条件为真，取 `expr1`
- 条件为假，取 `expr2`

例如：

```cpp
int max_value = a > b ? a : b;
```

这类写法适合做短小、直接的二选一。

## 什么时候别用三目运算符

如果表达式已经长到你需要反复确认真假分支，那通常就该换成 `if` / `else`。

例如教学上应该倾向于：

```cpp
if (a > b) {
    result = a;
} else {
    result = b;
}
```

当分支里还有额外逻辑时，这种写法往往更清楚。

## 易错点

- 明明手里是指针，却还写 `p.size()`。
- 忘记 `ptr->mem` 等价于 `(*ptr).mem`，从而漏掉必要括号。
- 把三目运算符写成多层嵌套，导致表达式难读。
- 只图简短，忽略 `if` / `else` 有时更适合表达复杂意图。
