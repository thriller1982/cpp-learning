# 表达式基础、优先级与求值顺序(expressions_basics_and_evaluation_order)

## 核心概念

- 表达式不只是“算出一个值”，它还有自己的类型和值类别。
- 左值通常有身份、能定位到对象；右值更像临时结果或纯值。
- 优先级决定怎么分组，结合律决定同级运算符怎么结合。
- 求值顺序不等于优先级，不要把“先绑定”理解成“一定先算”。
- 一旦表达式里带副作用，就应当优先拆开写，避免依赖顺序。

## 左值与右值

可以先这样抓重点：

- 左值：通常有名字、有身份、能表示一个对象
- 右值：通常是字面值、临时结果或纯值

例如：

```cpp
int i = 42;
i = 0;      // i 是左值
// 42 = i;  // 错误：42 不是左值
```

这里最值得记住的不是定义抠得多细，而是：

- 左值经常能放在赋值号左边
- 右值通常只能拿来参与计算或初始化

## 运算对象转换与结果类型

很多运算符对运算对象类型有要求。如果类型不完全匹配，编译器常常会先做转换。

```cpp
int i = 3;
double d = 4.5;
auto result = i + d;
```

这里：

- `i` 会先参与类型转换
- `result` 的类型是 `double`

这说明表达式本身也有类型：

```cpp
auto x = 3 + 4.5;   // x 是 double
```

## 优先级与结合律

优先级决定谁先“绑定”得更紧。

```cpp
int value = 3 + 4 * 5;
```

这行等价于：

```cpp
int value = 3 + (4 * 5);
```

结合律解决的是：当两个运算符优先级相同，该怎么分组。

赋值运算符通常是右结合：

```cpp
int a = 0;
int b = 0;
int c = 0;

a = b = c = 1;
```

这可以理解成：

```cpp
a = (b = (c = 1));
```

## 求值顺序不是优先级

这是这一章最容易混淆的地方。

```cpp
f() + g() * h();
```

这只能说明：

- `g() * h()` 这一部分会先分组
- 但不代表 `g()` 一定先于 `h()` 调用

所以一定要把这句话记牢：

优先级管分组，求值顺序管先算谁，它们不是一回事。

一旦表达式里还带副作用，就更不该赌顺序。

```cpp
int i = 0;
// i = ++i + i++;   // 危险：不要这样写
```

像这种写法，教学上要知道它为什么危险，但实际写代码时不要模仿。

## 常见的优先级感觉

不需要死背整张表，但常用层次要有感觉。

高优先级常见项：

- `()`
- `[]`
- `.`
- `->`
- 后置 `++`、后置 `--`

中间常见项：

- `* / %`
- `+ -`
- `< <= > >=`
- `== !=`
- `&& ||`

较低优先级常见项：

- `?:`
- `= += -= *= /= %=`
- `,`

真遇到拿不准的表达式，最稳妥的做法不是硬背，而是直接加括号。

## 易错点

- 把优先级和求值顺序当成同一件事。
- 看见 `a = b = c`，却没意识到赋值是右结合。
- 只关注“结果值”，忽略了表达式本身也有类型。
- 在一个表达式里多次修改同一个对象。
- 觉得“能编译”就说明这种复杂表达式值得写。
