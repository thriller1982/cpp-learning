# 递增与递减运算符(increment_and_decrement_operators)

## 核心概念

- 递增递减运算符包括前置 `++i`、`--i` 和后置 `i++`、`i--`。
- 前置是先改再用，后置是先用再改。
- 在迭代器上，通常优先写前置 `++it`。
- 不要在一个表达式里多次修改同一个对象。

## 前置与后置的区别

前置形式：

```cpp
int i = 0;
int j = ++i;   // i = 1, j = 1
```

这里是先把 `i` 改成 `1`，再把结果拿去初始化 `j`。

后置形式：

```cpp
int i = 0;
int j = i++;   // i = 1, j = 0
```

这里是先使用旧值，再完成递增。

## 为什么迭代器更常用前置 `++`

```cpp
for (auto it = nums.begin(); it != nums.end(); ++it) {
    // 使用 *it
}
```

书里更强调这种写法：

- `++it`
- 而不是 `it++`

原因可以先简单记成：后置往往需要保留旧值，理论上更低效。  
对初学阶段来说，更重要的是把这种风格记住。

## 不要把修改和使用搅在一起

递增递减本身就会改对象状态，所以一旦和别的操作混得太紧，就很容易出问题。

```cpp
int i = 0;
// cout << i << i++;   // 别这么写
```

教学上应当知道这种表达式危险，但实际代码里应该主动避开。

更清楚的写法通常是拆开：

```cpp
cout << i;
++i;
```

## 一个实用判断

如果你只是想“让它往前走一步”，那通常优先考虑前置。  
如果你明确需要旧值，那才使用后置。

## 易错点

- 把前置和后置当成完全等价。
- 遍历迭代器时默认写 `it++`，却没意识到教材更推荐 `++it`。
- 在一个表达式里既读取又多次修改同一个对象。
- 觉得写得短就一定更好，忽略可读性和求值风险。
