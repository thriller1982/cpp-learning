# 关系运算符与逻辑运算符(relational_and_logical_operators)

## 核心概念

- 关系运算符包括 `<`、`<=`、`>`、`>=`、`==`、`!=`。
- 逻辑运算符包括 `!`、`&&`、`||`。
- 这些表达式的结果通常可以看成布尔值。
- `&&` 和 `||` 有短路求值，这在判空时非常重要。
- `=` 是赋值，`==` 才是比较，这两个最容易写混。

## 关系运算符

```cpp
int a = 3;
int b = 5;

bool less = a < b;
bool same = a == b;
bool diff = a != b;
```

这类表达式的重点不是“得到 0 或 1”，而是它们表达的是一个真假判断。

## 逻辑运算符

```cpp
bool ok1 = true && false;
bool ok2 = true || false;
bool ok3 = !true;
```

逻辑运算通常用来把多个条件拼成一个更大的条件。

## 短路求值

`&&` 和 `||` 最关键的性质是短路。

```cpp
if (p && p->empty()) {
    // 只有 p 非空时，才会继续检查 p->empty()
}
```

这里如果 `p` 为假，右边就不会继续求值。  
这不是“小优化”，而是写安全条件判断时非常常用的机制。

`||` 也一样：

```cpp
if (ready || retry()) {
    // 如果 ready 已经为真，就不会再调用 retry()
}
```

## 不要把 `=` 和 `==` 混了

```cpp
if (i == 0) {
    // 比较 i 是否等于 0
}
```

而下面这种是常见错误：

```cpp
// if (i = 0) { }   // 错误含义：这是赋值，不是比较
```

问题不只是“少写了一个等号”，而是整个表达式语义都变了。

## 条件表达式里的真假

在 `if`、`while` 这类位置，表达式最终会按真假来理解。

```cpp
if (count) {
    // count 非 0 时进入
}
```

不过教学上知道它能这么写就够了。只要表达式已经不够直观，就应优先写得更明确。

## 易错点

- 把 `=` 写成 `==`，或者反过来。
- 忘记 `&&` 和 `||` 是短路运算。
- 写指针成员访问时，先用 `p->member`，却没先确认 `p` 是否有效。
- 把关系运算和逻辑运算混成一长串，却不加括号澄清意图。
