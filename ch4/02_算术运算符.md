# 算术运算符(arithmetic_operators)

## 核心概念

- 算术运算符包括 `+`、`-`、`*`、`/`、`%`。
- 整数除法会截断小数部分，不会自动得到浮点结果。
- 取余运算符 `%` 只用于整数类型。
- 混合算术类型一起运算时，编译器常常会先做类型提升或转换。

## 基本算术运算

```cpp
int a = 6 + 2;
int b = 6 - 2;
int c = 6 * 2;
int d = 6 / 2;
int e = 6 % 4;
```

前四个都比较直观，最容易出错的是 `/` 和 `%`。

## 整数除法会截断

```cpp
int x = 5 / 2;   // 结果是 2
```

这里不是四舍五入，也不是自动变成 `2.5`，而是直接丢掉小数部分。

如果你想得到浮点结果，至少要让一个运算对象是浮点类型：

```cpp
double y = 5 / 2.0;   // 结果是 2.5
```

## 取余只用于整数

```cpp
int r = 5 % 2;   // 结果是 1
```

但下面这种写法不成立：

```cpp
// double bad = 5.0 % 2.0;
```

原因不是语法偏好，而是 `%` 本来就不是给浮点数设计的。

## 混合类型运算

```cpp
int i = 3;
double d = 2.5;
auto result = i + d;
```

这里：

- `i` 会先转换
- `result` 的类型是 `double`

也就是说，表达式最终结果类型不一定和左边那个变量一样。

## 算术运算里的一个常见提醒

只要表达式里混进了不同类型，就不要只盯着字面上的符号，还要同时问自己：

- 运算前发生了什么转换
- 结果类型是什么
- 有没有信息丢失

这在 `int` 和 `double` 混算时尤其常见。

## 易错点

- 误以为 `5 / 2` 的结果是 `2.5`。
- 把 `%` 用在浮点数上。
- 只看变量原始类型，不看表达式最终类型。
- 混合运算后把结果塞回更小类型时，忽略可能的信息丢失。
