# 赋值运算符(assignment_operators)

## 核心概念

- 最基本的赋值运算符是 `=`，复合赋值包括 `+=`、`-=`、`*=`、`/=`、`%=` 等。
- 赋值号左边必须是可修改左值。
- 赋值表达式本身也有结果，所以可以连写。
- 赋值时如果类型不匹配，往往会先发生转换。

## 基本赋值

```cpp
int i = 0;
i = 42;
```

这里左边的 `i` 是一个可修改左值，所以赋值合法。

而下面这种不合法：

```cpp
// 42 = i;   // 错误：左边不是可修改左值
```

## 赋值表达式本身也有结果

```cpp
int a = 0;
int b = 0;

a = b = 0;
```

这之所以成立，是因为：

- `b = 0` 本身就是一个表达式
- 这个表达式的结果还能继续参与下一次赋值

所以这行可以理解成：

```cpp
a = (b = 0);
```

## 复合赋值

```cpp
int count = 10;
count += 1;
count *= 2;
```

这些写法通常比下面更直接：

```cpp
count = count + 1;
count = count * 2;
```

当你的意图就是“在原值基础上更新”，复合赋值往往更自然。

## 赋值中的类型转换

```cpp
double d = 3.14;
int i = 0;

i = d;   // i 变成 3
```

这里不是神秘规则，而是：

- 右侧值会先转换成左侧对象的类型
- 小数部分会被丢掉

这说明赋值不只是“把右边搬过去”，它还可能伴随信息丢失。

## 赋值表达式要尽量写清楚

像下面这种连写，语义还算清楚：

```cpp
int x = 0;
int y = 0;
int z = 0;

x = y = z = 0;
```

但如果赋值和别的运算混在一起太多，可读性就会迅速下降。  
一旦你发现自己需要停下来重新读表达式，就该考虑拆开写。

## 易错点

- 忘记赋值左边必须是可修改左值。
- 把赋值表达式误以为“没有返回值”。
- 把 `i += 1` 和 `i = i + 1` 的语义环境混着看，却没注意类型转换。
- 赋值时从 `double` 到 `int`，却忽略小数部分会丢失。
