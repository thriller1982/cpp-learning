# 参数传递(parameter_passing)

## 核心概念

- 参数传递的关键在于：形参和实参是否是同一个对象。
- 传值会拷贝出一份副本，传引用会直接绑定到实参。
- 大对象如果只读，通常优先用 `const T&`。
- 数组作为函数参数时，通常会退化成指针。
- 可变参数函数最常见、最自然的方式是 `initializer_list`。

## 值参数

传值的意思是把实参拷贝一份给形参：

```cpp
int reset(int x) {
    x = 0;
    return x;
}
```

这里函数里修改的是 `x` 这份副本，不会影响调用者原来的对象。

```cpp
int i = 10;
reset(i);
// i 仍然是 10
```

传值通常适合以下情况：

- 参数很小，拷贝成本低
- 函数不需要修改实参
- 函数内部需要一份独立副本

## 引用参数

引用参数不会产生新的对象副本，而是直接给实参起一个别名：

```cpp
void reset(int& i) {
    i = 0;
}
```

这里修改 `i`，就是直接修改调用者传入的那个对象。

```cpp
int val = 10;
reset(val);
// val 现在变成 0
```

因此引用参数的关键点是：函数操作的就是原对象，而不是副本。

## 传参选择与 `const` 引用

如果参数是 `string`、`vector` 这类较大的对象，按值传递往往意味着额外拷贝。  
这时如果函数只需要读取参数，通常更适合写成 `const` 引用：

```cpp
bool isShorter(const string& s1, const string& s2) {
    return s1.size() < s2.size();
}
```

这种写法有几个好处：

- 不拷贝大对象
- 明确保证函数不会修改参数
- 能接受 `const` 对象，也能接受临时对象

可以把它概括成一句话：大对象只读时，优先考虑 `const T&`。

## 数组形参

数组参数是函数章节中的高频难点。  
最先要记住的是：数组不能像普通对象那样按值整体传给函数。

```cpp
void print(const int* beg, const int* end);
void print(const int ia[], size_t size);
```

这两种写法看起来不同，但本质上都不是传整个数组副本。

在函数参数里：

```cpp
void f(int arr[]);
```

本质上等价于：

```cpp
void f(int* arr);
```

因此函数无法仅靠这个形参知道数组长度。  
数组参数常见的处理方式有：

- 额外再传一个长度
- 传首尾指针
- 传固定大小数组的引用

如果参数必须是固定大小的数组，可以写成：

```cpp
void f(int (&arr)[10]);
```

这里保留了数组大小信息，但写法也更严格。

## 特殊形参形式

`main` 常见的一种写法是：

```cpp
int main(int argc, char* argv[])
```

其中：

- `argc` 表示命令行参数个数
- `argv` 表示参数字符串数组

例如运行：

```cpp
prog hello world
```

通常 `argv[0]` 是程序名，`argv[1]` 是 `"hello"`，`argv[2]` 是 `"world"`。

如果函数参数个数不固定，更推荐的现代写法是 `initializer_list`：

```cpp
void error_msg(initializer_list<string> il) {
    for (const auto& msg : il)
        cout << msg << " ";
}
```

调用时可以写：

```cpp
error_msg({"functionX", "expected", "actual"});
```

相比老式的省略符 `...`，`initializer_list` 更自然，也保留了明确的类型信息。

## 易错点

- 把传值误以为传引用，以为修改形参就会影响实参。
- 该用 `const string&` 的地方直接写 `string`，造成不必要拷贝。
- 忘记引用形参会直接影响调用者传入的对象。
- 误以为数组参数还能自动保留完整数组长度信息。
- 看到 `void f(int arr[])`，却没意识到本质上是指针参数。
