# 函数基础(function_basics)

## 核心概念

- 函数用来把可复用逻辑封装起来。
- 函数体里的对象和形参都属于局部对象。
- 普通局部对象每次调用都会重新创建，局部静态对象只初始化一次。
- 函数可以先声明，后定义。
- 多文件程序依赖“声明可见、定义可分离编译”的组织方式。

## 这一章在讲什么

第 6 章的主线是函数。  
这一章主要讨论函数如何声明与定义、参数如何传递、结果如何返回，以及重载、匹配和函数指针这些更进一步的用法。

## 局部对象与形参

函数体内部定义的对象叫局部对象：

```cpp
int f() {
    int x = 0;
    return x;
}
```

这里的 `x` 只在 `f` 的函数体中有效，函数结束后就离开作用域。

形参本身也属于局部对象：

```cpp
int fact(int val) {
    int ret = 1;
    while (val > 1)
        ret *= val--;
    return ret;
}
```

这里的 `val` 和 `ret` 一样，都只属于当前这次函数调用。  
调用开始时创建，调用结束时销毁。

## 自动对象与局部静态对象

普通局部变量通常是自动对象，进入块时创建，离开块时销毁：

```cpp
void g() {
    int i = 0;
    {
        int j = 42;
    }
    // 这里 j 已经不存在了
}
```

如果在函数里把局部变量声明成 `static`，它就不再是每次调用都重新创建的对象：

```cpp
size_t count_calls() {
    static size_t ctr = 0;
    return ++ctr;
}
```

`ctr` 只初始化一次，之后会在整个程序运行期间一直存在，并保留上一次调用后的值。

## 函数声明与定义

函数可以先声明，后定义：

```cpp
int fact(int val);
```

这条声明告诉编译器函数名、参数类型和返回类型。  
真正的定义可以稍后再写：

```cpp
int fact(int val) {
    int ret = 1;
    while (val > 1)
        ret *= val--;
    return ret;
}
```

声明和定义的区别在于：

- 声明说明函数长什么样
- 定义真正给出函数体

只要调用点已经见过声明，编译器就知道该如何处理这个调用。

## 分离式编译

大型程序通常不会把所有代码写在同一个文件中。  
更常见的做法是：

- 头文件放声明
- 源文件放定义
- 各个源文件分别编译
- 最后再链接到一起

它的关键点是：调用函数时，当前文件只需要先看到声明，不必先看到定义本身。

## 易错点

- 把形参忘记成局部对象的一部分。
- 误以为普通局部变量会保留上一次调用留下的值。
- 不清楚 `static` 局部对象和普通局部对象的生命周期差异。
- 把函数声明和函数定义混成一回事。
- 调用别处定义的函数时，忘记当前文件必须先看到声明。
