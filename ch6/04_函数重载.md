# 函数重载(function_overloading)

## 核心概念

- 重载允许同一个函数名对应多个不同参数列表。
- 区分重载靠的是参数列表，不是返回类型。
- 仅返回类型不同，不能构成合法重载。
- 名字查找先发生，重载选择后发生。
- 内层同名函数可能会把外层重载集隐藏掉。

## 重载的基本要求

重载的基本含义是同一个名字对应多组不同参数：

```cpp
void print(int);
void print(double);
void print(const string&);
```

调用时，编译器会根据实参选择最合适的那个版本：

```cpp
print(42);
print(3.14);
print("hello");
```

能否构成重载，关键看参数列表是否不同，包括：

- 参数个数
- 参数类型
- 参数顺序

## 参数列表与 `const`

仅返回类型不同，不算合法重载：

```cpp
void f(int);
int f(int);
```

原因是调用函数时，编译器不能只靠你想接收什么返回类型来区分版本。

对传值参数来说，顶层 `const` 通常也不能单独区分重载：

```cpp
void f(int);
void f(const int);
```

这两条声明通常不构成两个不同的重载，因为传值参数拿到的本来就是副本。

## 名字查找与作用域

重载和作用域放在一起时，要先记住一个顺序：

- 先做名字查找
- 再做重载选择

这意味着如果当前作用域已经有同名函数，外层作用域的同名版本可能根本不会参与匹配。

例如：

```cpp
void print(int);
void print(double);

void f() {
    void print(const string&);
    print("hello");
}
```

在 `f` 的内部，内层这个 `print` 会把外层那组同名函数隐藏掉。  
这里不是把三个版本放到一起重载，而是名字查找已经先把外层那组挡住了。

## 重载的价值与边界

重载的价值不只是少起几个名字，而是把同一类操作统一到同一个接口名下，让调用点更自然。

不过前提是这些重载版本的语义确实属于同一类操作。  
如果只是参数不同、语义却差得很远，重载反而会让接口更难理解。

## 易错点

- 误以为返回类型不同就能构成重载。
- 只看函数名相同，就以为一定属于同一个重载集。
- 忘记名字查找先于重载选择。
- 不清楚内层同名函数会隐藏外层重载集。
- 用重载堆出一组语义并不一致的接口，反而让调用点更难理解。
