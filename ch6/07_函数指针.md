# 函数指针(function_pointers)

## 核心概念

- 函数名在很多场景下可以当成值来使用。
- 函数指针可以指向某种特定签名的函数。
- 函数指针常用于把函数传给别的函数，或把函数保存起来以后再调用。
- 调用函数指针时，写 `pf(args)` 和 `(*pf)(args)` 都可以。
- 看函数指针声明时，先抓住名字，再看 `*` 和参数列表。

## 函数指针的声明

先看一个普通函数声明：

```cpp
bool lengthCompare(const string&, const string&);
```

如果要定义一个指针，专门指向这种参数列表和返回类型的函数，可以写：

```cpp
bool (*pf)(const string&, const string&) = lengthCompare;
```

这里：

- `pf` 是变量名
- `(*pf)` 说明它是指针
- 后面的参数列表说明它能指向哪类函数

函数指针要求参数列表和返回类型都匹配。

## 函数名与函数指针

下面这种写法：

```cpp
pf = lengthCompare;
```

并不是在调用函数，而是在把函数本身作为一个值使用。  
这也是函数指针的重要意义之一：函数不只是能被调用，还能被引用、传递和保存。

## 调用与用途

函数指针有两种常见调用方式：

```cpp
pf("a", "bb");
(*pf)("a", "bb");
```

两种写法效果相同，多数时候直接写 `pf(...)` 就够了。

函数指针常见的用途包括：

- 把函数作为参数传给别的函数
- 把函数保存起来，之后再调用
- 在多种处理策略之间切换

它的价值不在于语法本身复杂，而在于可以把行为也当成数据一样传来传去。

## 读取函数指针声明

函数指针声明看起来容易乱，但读法其实比较固定：

```cpp
bool (*pf)(const string&, const string&);
```

可以按下面的顺序拆开看：

- 先找名字：`pf`
- 再看它是不是指针：`(*pf)`
- 最后看它指向什么函数：参数是两个 `const string&`，返回值是 `bool`

把这个顺序记住后，函数指针声明会清楚很多。

## 易错点

- 把函数名当成立即调用，而不是把它看成可被引用的函数实体。
- 看函数指针声明时，不先抓名字和括号结构，结果越看越乱。
- 不清楚函数指针要求参数列表和返回类型都匹配。
- 以为 `pf(args)` 和 `(*pf)(args)` 是两种不同语义。
- 一看到函数指针语法复杂，就忽略了它的实际用途。
